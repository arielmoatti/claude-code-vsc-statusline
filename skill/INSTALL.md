# INSTALL - אשף התקנת הסקיל statuswatch

**מטרת הקובץ הזה:** מנהל את תהליך ההתקנה של סקיל statuswatch. קורא: Claude (המודל שמבצע את ההתקנה עבור המשתמש).

## כלל על (אל תעבור עליו)

**כל התקשורת עם המשתמש - בעברית.** טוקנים טכניים (שמות קבצים, נתיבים, קוד) נשארים באנגלית בתוך הטקסט העברי.

**אל תכתוב כלום לדיסק עד שהמשתמש אישר.** לפני כתיבת הקובץ הסופי - הצג לו סיכום של הבחירות וקבל אישור.

---

## Phase 0 - פתיחה ובדיקת התלות

הצג פתיחה קצרה בעברית:

> ברוך הבא להתקנת statuswatch - סקיל שמלווה עבודה כבדה של Claude Code (הרבה סוכנים, workflows ארוכים) ועוצר אותה **לפני** שמכסת ה-5 שעות נגמרת, כדי שתמיד יישאר לך מרווח לעבוד. אני אשאל אותך שתי שאלות קצרות ואכתוב את הסקיל מותאם אליך. מתחילים?

חכה לאישור. אחר כך **בדוק את התלות** - הסקיל עובד אך ורק בסינרגיה עם תוסף ה-statusline (הוא החיישן היחיד):

```bash
node -e "const p=require('path').join(require('os').tmpdir(),'claude','statusline-usage-cache.json'); try{const c=require(p); console.log('OK, age='+Math.round((Date.now()-c.fetchedAt)/1000)+'s')}catch(e){console.log('MISSING')}"
```

- **`MISSING` או age של שעות** ← תוסף ה-statusline לא מותקן/לא רץ. עצור והסבר: "הסקיל תלוי בתוסף Claude Code Statusline (מאותו ריפו). התקן אותו קודם לפי ה-README, ודא שהשורה מופיעה בשורת הסטטוס, וחזור אליי." אל תמשיך בהתקנה.
- **`OK` עם age סביר (עד כמה דקות)** ← ממשיכים.

## Phase 1 - שאלות ההתאמה (חובה דרך AskUserQuestion)

חובה להשתמש בכלי `AskUserQuestion` (חלונית popup מובנית של Claude Code), **לא** בטקסט חופשי. אם הכלי לא זמין ברשימה הישירה, קרא קודם ל-`ToolSearch` עם `select:AskUserQuestion` כדי לטעון את הסכמה.

### שאלה 1 - תקרת הניצול (CAP)

> עד כמה לנצל את מכסת ה-5 שעות לפני עצירת העבודה? מה שנשאר מעבר לתקרה הוא המרווח שלך - להמשיך לעבוד במקביל, לעצור סשן, או סתם לא להיתקע.

אפשרויות (בחירה אחת):
- **85% (מומלץ)** - ניצול מקסימלי של החלון עם מרווח ביטחון סביר
- **90%** - אגרסיבי; מרווח מינימלי, למי שמריץ לילות שלמים ללא עבודה מקבילה
- **75%** - שמרני; מרווח נוח לעבודה מקבילה כבדה
- **60%** - מרווח גדול מאוד; העבודה הכבדה תתקדם לאט יותר בין איפוסים

### שאלה 2 - ביטויי הפעלה בעברית (multiSelect)

> באילו ביטויים טבעיים תרצה להפעיל את הסקיל תוך כדי שיחה? בחר את כל מה שמתאים לסגנון שלך (אפשר גם להוסיף משלך ב-Other):

אפשרויות (multiSelect):
- **"שמור לי על המכסה"**
- **"שים עין על הסטטוס"**
- **"אל תשרוף לי את החלון"**
- **"תשמור על התקציב"**

### שאלה 3 - ביטויי הפעלה באנגלית (multiSelect)

> ואותו דבר באנגלית - למקרה שאתה עובד גם באנגלית:

אפשרויות (multiSelect):
- **"watch the quota"**
- **"mind the budget"**
- **"keep an eye on usage"**
- **"don't burn the window"**

**הערה:** המילה "statuswatch" עצמה נרשמת תמיד כטריגר, בלי קשר לבחירות.

## Phase 2 - בניית הקובץ וכתיבתו

1. משוך את התבנית:

```
https://raw.githubusercontent.com/arielmoatti/claude-code-vsc-statusline/HEAD/skill/statuswatch.md
```

2. בצע את ההחלפות:
   - `{{CAP}}` ← המספר שנבחר בשאלה 1 (למשל `85`).
   - `{{TRIGGER_PHRASES}}` ← כל הביטויים שנבחרו בשאלות 2-3 (כולל תוספות חופשיות מ-Other), מופרדים בפסיקים, כל ביטוי במרכאות. למשל: `"שמור לי על המכסה", "שים עין על הסטטוס", "watch the quota",`.
3. הצג למשתמש סיכום קצר בעברית: התקרה שנבחרה + רשימת הטריגרים + היעד `~/.claude/skills/statuswatch/SKILL.md`. קבל אישור.
4. אם קיים כבר `~/.claude/skills/statuswatch/SKILL.md` - שאל (שוב דרך AskUserQuestion) אם להחליף, ורק אז כתוב.
5. כתוב את הקובץ ל-`~/.claude/skills/statuswatch/SKILL.md` (צור את התיקייה אם חסרה).

## Phase 3 - סיכום והסבר "עבודה במשמרות"

דווח שההתקנה הושלמה (נתיב הקובץ, התקרה, הטריגרים), ואז הסבר בעברית את יכולת ההמשכיות - חשוב שהמשתמש ידע שהיא קיימת, כי היא מופעלת רק בהוראה שלו:

> דבר אחרון ששווה להכיר - **עבודה במשמרות**. כשאתה נותן ל-Claude משימה גדולה שלא נכנסת בחלון 5 שעות אחד, אתה יכול לבקש (למשל: "עבוד במשמרות עד שתסיים") ואז הוא: עובד עד התקרה שקבעת ← עוצר ← ממתין לאיפוס המכסה (הוא יודע מתי בדיוק מה-statusline) ← מוודא שהמכסה באמת התאפסה ← וממשיך מאותה נקודה. וחוזר חלילה, עד השלמת המשימה - "טייס אוטומטי" לכל הלילה.
>
> ברוב המשימות אין בזה צורך - הן נגמרות בתוך חלון אחד. אבל למשימות ענק (סבב QA מלא, מיגרציה רחבה, מחקר עמוק) זה משנה הכל: אתה הולך לישון, והעבודה ממשיכה לבד בלי לחסום אותך.

סיים עם בדיקה מהירה:

> נסה עכשיו: כתוב לי "{אחד הטריגרים שנבחרו}" ותראה שאני קורא את מצב המכסה ומדווח.
